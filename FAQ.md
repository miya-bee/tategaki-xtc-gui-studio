# よくある質問

## このアプリは何をするものですか？

EPUB、TXT、Markdown などのテキストを、縦書き XTC / XTCH 形式へ変換するための Python GUI アプリです。

主な対象は、青空文庫系の小説本文や、縦書きで読みたいプレーンテキストです。

## どの形式を入力できますか？

主に次の形式を想定しています。

- EPUB
- TXT
- Markdown
- 画像系入力

ただし、Markdown や README のような横書き前提の文書は、記法や半角記号がそのまま表示されるため、縦書きでは不自然に見える場合があります。

## Python が必要ですか？

はい。v1.2.2 の公開対象は Python GUI版です。

Python 3.10 / 3.11 / 3.12 のいずれかを用意してください。迷った場合は Python 3.12 を推奨します。

## exe版はありますか？

このリポジトリの公開対象は Python GUI版です。exe版を別途配布する場合は、配布元の案内に従ってください。

## フォントによって記号の位置が違います

縦書き表示の見え方は、使用するフォントに大きく依存します。

特に次のような記号は、フォントごとの差が出やすいです。

```text
～ 〜 〰 ~ ∼ ∽ ∿ ≀
、 。 」 』 一
```

v1.2.2 では、句読点、漢数字「一」、下鍵括弧、波線描画、波線位置をGUIから調整できます。まずは標準設定で使い、気になる箇所だけ補正してください。

## 波線描画の「回転グリフ」と「別描画」は何が違いますか？

`回転グリフ` は、フォントの記号を回転して縦書き内に配置する方式です。

`別描画` は、波線をアプリ側で別途描画する方式です。

フォントによって自然に見える方式が異なるため、見た目を確認しながら選んでください。迷った場合は、初期値の `回転グリフ` をおすすめします。

## 波線位置は何を選べばよいですか？

迷った場合は `標準` のままで構いません。

波線が上寄りに見える場合は、`下補正弱` または `下補正強` を試してください。

## 半角コンマやピリオドが左下に寄って見えます

横書き前提の半角記号は、縦書き表示で不自然に見える場合があります。

例:

```text
, .
```

これは特に README、Markdown、英文、URL、ファイル名、バージョン番号などで目立ちます。

v1.2.2 では、青空文庫系の小説本文を主対象としているため、半角コンマ・ピリオドの自動変換は行っていません。必要に応じて、入力テキスト側で全角記号や和文句読点に置き換えてください。

## Markdown がきれいに縦書きになりません

Markdown は横書き前提の記法を多く含むため、縦書き本文としては不自然になる場合があります。

例:

```text
# 見出し
- 箇条書き
`code`
https://example.com
README.md
```

小説本文のように読むことを目的とする場合は、Markdown記法を取り除いたプレーンテキストにしてから変換することをおすすめします。

## プレビューと実際の表示が少し違います

プレビューは変換結果の目安です。

最終的な見え方は、使用フォント、閲覧環境、画面サイズ、表示アプリ側の処理によって変わる場合があります。

## どの設定を触ればよいか分かりません

まずはプリセットを選び、標準設定のままプレビューしてください。

気になる箇所がある場合だけ、以下を調整します。

- 記号がずれる: 句読点 / 漢数字一 / 下鍵括弧
- 波ダッシュやチルダが不自然: 波線描画 / 波線位置
- 文字が大きい・小さい: 本文サイズ
- 行が詰まる・広すぎる: 行間
- ページ内の余白が気になる: 余白

## Font フォルダは必要ですか？

release zip に Font フォルダが同梱されている場合は、そのまま使えます。

Font フォルダがない場合でも、利用可能な system font へフォールバックします。ただし、表示やテスト結果が環境によって変わる場合があります。

## 同梱フォントのライセンスは？

同梱フォントは SIL Open Font License 1.1 に基づいて配布されています。詳細は `LICENSE_OFL.txt` を参照してください。

アプリ本体とフォントのライセンスは別です。
