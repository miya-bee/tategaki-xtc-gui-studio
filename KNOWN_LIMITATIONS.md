# 既知の注意点・仕様

この文書は、縦書きXTC GUI Studio v1.3.1 の既知の注意点をまとめたものです。

## 1. 記号類の見え方はフォントに依存します

縦書き表示では、フォントごとのグリフ形状や基準位置の違いが見た目に大きく影響します。

特に次の文字は、フォントによって位置や形が変わりやすいです。

```text
～ 〜 〰 ~ ∼ ∽ ∿ ≀
、 。 」 』 一
```

v1.3.1 では補正設定を用意していますが、すべてのフォントで完全に同じ見え方にすることはできません。

## 2. 横書き前提の文書は不自然に見える場合があります

このアプリの主対象は、青空文庫系の小説本文やプレーンテキストです。

README、技術文書、Markdown、URL、コードブロック、表などは横書き前提で作られているため、縦書き表示では不自然になることがあります。

例:

```text
README.md
v1.3.1
https://example.com
file_name.txt
A, B, C.
```

## 3. 半角コンマ・半角ピリオドは補正対象外です

半角コンマ `,` や半角ピリオド `.` は、フォントによって縦書き内で左下寄りに見える場合があります。

v1.3.1 でも、これらを自動で全角記号や和文句読点へ変換していません。

理由:

- URL、ファイル名、バージョン番号、小数、桁区切りなどに多く使われる
- 自動置換すると、文書の意味や見た目を壊す可能性がある
- 主対象である青空文庫系の小説本文では、大きな問題になりにくい

必要な場合は、入力テキスト側で事前に置換してください。

## 4. Markdown記法の整形は限定的です

Markdown ファイルを読み込むことはできますが、Markdownを美しく組版する専用エンジンではありません。

次のような記法は、そのまま表示される場合があります。

```text
# 見出し
- 箇条書き
* 強調 *
`code`
```code block```
```

本文として読みたい場合は、Markdown記法を整理したプレーンテキストへ変換してから使うことをおすすめします。

## 5. プレビューは最終表示の完全再現ではありません

GUIプレビューは、変換結果を確認するための目安です。

最終的な表示は、次の要素で変わる場合があります。

- 使用フォント
- 画面サイズ
- 閲覧環境
- XTC / XTCH を表示する側の処理
- 白黒反転などの表示設定

## 6. 補正設定は万能ではありません

句読点、漢数字「一」、下鍵括弧、波線描画、波線位置の補正は、見た目を調整するためのものです。

ただし、フォントごとの差を完全に吸収するものではありません。違和感がある場合は、フォントの変更も試してください。

## 7. フォルダ一括変換の対象形式には依存関係があります

フォルダ一括変換では、TXT / Markdown などのテキスト系に加え、EPUB や画像系入力も対象にできます。

ただし EPUB は optional dependency が必要です。`ebooklib` などが不足している環境では、変換開始前に不足ライブラリの案内が表示されます。

## 8. 既存ファイルの扱いは事前に確認してください

フォルダ一括変換では、既存ファイルがある場合の扱いとして、スキップ、別名保存、上書きを選べます。

公開前後の確認では、まず `スキップ` を選ぶと安全です。上書きを選ぶ場合は、出力先フォルダを十分確認してください。


## 9. フォルダ一括変換の停止は即時強制終了ではありません

フォルダ一括変換中に停止ボタンを押すと、停止要求を受け付け、現在処理中のファイルが終わりしだい停止します。

これは、出力途中のファイル破損や処理状態の不整合を避けるためです。停止完了時には、成功・スキップ・失敗・処理済み件数に加えて、未処理件数が表示されます。
