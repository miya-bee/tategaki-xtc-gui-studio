# RELEASE NOTES v1.3.4

v1.3.4 は、GitHub 公開済みの v1.3.3 / v1.3.3.0 から、v1.3.3.x 内部作業版で積み重ねた改良をまとめた公開版です。

今回の更新は、変換形式そのものを変える大改造ではなく、縦書き表示の細かな調整、プレビュー操作、変換完了後の確認、中止処理、プリセット保存の安全性を中心にした改善です。

## 主な追加・改善

### ルビ消しモード

画像処理セクションに「ルビ消し」を追加しました。チェックすると、親文字は残したままルビだけを非表示にして変換できます。通常時は従来どおりルビ付きで表示します。

ルビ消しはプレビューだけでなく XTC / XTCH 保存にも反映されます。既存のルビ描画処理本体は大きく変更せず、描画前の入力データ側でルビ情報を外す方式にしています。

### 半角数字・半角英字・ASCII記号の縦書き表示調整

半角数字の縦位置を5段階で補正できる「半角数字」設定を追加しました。対象は半角数字のみで、全角数字やルビ内の数字は対象外です。半角数字の縦中横にも補正が反映されます。

半角数字の縦中横上限も、4文字 / 3文字 / 2文字 / 無し から選択できるようにしました。デフォルトは 2文字です。

半角英字やASCII記号は、縦中横扱いにせず1文字ずつ中央揃えで描画します。ASCII括弧類 `() [] {} <>` は右90度回転して中央揃えで表示します。

### プレビュー表示の改善

右ペインのフォントビュー、実機ビュー、実寸近似表示で、高倍率表示時の寄せ方と横スクロール挙動を調整しました。倍率を上げたときに急に左寄せへ切り替わったり、左側に不自然な空白が残ったりする問題を改善しています。

起動直後にフォントビューから実機ビューへ切り替えた際、実機表示だけが左へ寄る問題も修正しました。

### 画面レイアウトとヘルプ表示の整理

設定項目が増えたため、左ペインと右ペインの表示を整理しました。

- 余白表示順を「上 / 下 / 左 / 右」に統一
- 画像処理セクションの位置補正項目を2行に整理
- フォント選択行の「参照」ボタン位置を調整
- ヘルプ「？」の位置や説明文を整理
- 右ペイン下部の「ボタン反転」を歯車メニュー内の「ページ送りキー反転」へ移動
- 左右ペインの中央セパレーター既定位置を 600px に調整

### 変換対象欄へのドラッグ＆ドロップ

上部の変換対象欄へ、EPUB / TXT / Markdown / 画像などのソースファイルをドラッグ＆ドロップできるようにしました。ドロップ後は従来のファイル選択と同じ経路で、対象欄更新、ステータス更新、UI状態保存、プレビュー更新予約を行います。

### 変換完了後の確認導線

変換完了後に Windows エクスプローラーを自動起動しないようにしました。代わりに、右ペイン上部へ変換完了カードを表示し、保存件数や保存先の概要を確認できるようにしています。

完了カードの操作は、基本的に「保存先を開く」と「閉じる」に整理しました。エクスプローラーはユーザーが「保存先を開く」を押した場合だけ開きます。

### フォルダ一括変換の中止処理

フォルダ一括変換中の中止ボタンの挙動を改善しました。

中止時は現在処理中の個別変換 worker にも停止要求を伝え、可能な箇所では現在ファイルの処理中でも停止しやすくしました。中止は通常の失敗ではなく「中止」として集計・表示します。

EPUB変換中に中止した場合に「EPUB の本文描画中に失敗しました」「HTML / CSS が特殊な可能性があります」といったエラー表示になる問題も修正しました。中止後にプログレスバーが動き続ける問題、中止後に何も表示されない問題も修正しています。

### プレビュー更新の安定化

ルビ消し ON/OFF 時に、プレビューページが1ページ目へ戻らないようにしました。ページ数が減った場合だけ、存在するページ範囲内へ丸めます。

ディザリング ON/OFF などの設定変更時、プレビュー生成中の更新予約が積み重なって暴走する問題も修正しました。

### プリセット保存の安全性

初回起動時など、`tategakiXTC_gui_studio.ini` がまだ存在しない状態からプリセット保存を行った場合でも、保存結果を確認するようにしました。

プリセット保存後に `QSettings.sync()` を実行し、保存状態を確認します。さらに、保存した値を再読込して、書き込んだ値と一致するか検証します。書き込み不可フォルダ、読み取り専用 ini、zip 内からの直接起動などで保存できない場合は、成功表示ではなく警告を出します。

## GitHub / release 配布まわり

- GitHub Actions 用の最小 CI を維持しています。
- source-only zip では `.github/workflows/python-tests.yml` を含めます。
- public release zip では `.github/` を除外します。
- handoff md、logs、cache、test output などの開発用生成物が zip に混入しないよう整理しました。
- README に、このプロジェクトが Source Available であり、OSI認定の Open Source ではないことを明記しました。

## 変更していない主な部分

以下の中核部分は、v1.3.3 から大きく変更していません。

- XTC / XTCH 保存形式
- 既存のルビ描画ルーチン本体
- 基本的な縦書き描画ロジック
- 既存の句読点・漢数字一・下鍵括弧補正の基本仕様

## 既知の注意

- PDF入力対応、PDF出力、CBZ出力はまだ含まれていません。
- プリセット名編集や、プリセット選択時に設定欄へ値を自動反映する機能はまだ含まれていません。
- release zip にフォントを含める場合は、`Font/` フォルダと `LICENSE_OFL.txt` の整合を確認してください。

詳細な変更履歴は `CHANGELOG.md` も参照してください。
