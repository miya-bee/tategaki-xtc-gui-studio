# RELEASE NOTES v1.3.5

v1.3.5 は、公開版 v1.3.4 の表示改善をまとめた小修正版です。

主な対象は、note コメント欄で報告された縦中横まわりの見た目の違和感です。XTC / XTCH 保存形式、ルビ描画の基本仕様、フォルダ一括変換などの中核処理は変更していません。

## 主な修正・改善

### 縦中横記号ペアの表示改善

縦書き本文中の記号ペア表示を調整しました。

- 半角 `??` を、`!!` / `!?` / `?!` と同じく記号ペアとして扱うようにしました。
- 半角 `!?` / `??` / `!!` / `?!` を、1文字ずつではなく2文字ペアとして扱います。
- 全角 `！？` / `？？` / `！！` / `？！` の右方向への張り出しを抑えるため、縦中横描画の貼り付け位置を ink bbox 中心基準へ寄せました。
- v1.3.4.3 で試した半角記号ペアの強い縮小は撤回し、半角 `!?` などが小さくなりすぎないようにしました。
- 手動の「プレビュー更新」時に、未処理の設定変更由来 live preview 予約をキャンセルし、プレビュー更新が連鎖する場合がある問題を修正。

### 縦中横記号の縦位置補正

画像処理セクションの位置補正群に「縦中横記号」を追加しました。

対象は以下の記号ペアのみです。

- 全角: `！？` / `？？` / `！！` / `？！`
- 半角: `!?` / `??` / `!!` / `?!`

補正モードは既存の位置補正と同じ5段階です。

- 下補正 強
- 下補正 弱
- 標準
- 上補正 弱
- 上補正 強

デフォルトは `標準` です。Noto 系フォントで記号ペアが上寄りに見える場合は、まず `下補正 弱` から確認してください。

### 既存補正との関係

今回の「縦中横記号」補正は、記号ペア専用です。

以下の既存項目は従来どおり分離されています。

- 句読点補正
- 漢数字「一」補正
- 半角数字補正
- 下鍵括弧補正
- 波線描画 / 波線位置

半角数字の縦中横（例: `88` / `123` / `2026`）には、今回の「縦中横記号」補正は適用されません。

## GitHub / release 配布まわり

- v1.3.5 は、v1.3.4 のパッチリリースとして扱います。
- GitHub Release tag は `v1.3.5` を想定しています。
- Release 本文にはこの `RELEASE_NOTES_v1_3_5.md` の内容を使用できます。
- 詳細な変更履歴は `CHANGELOG.md` も参照してください。

## 変更していない主な部分

以下は v1.3.4 から大きく変更していません。

- XTC / XTCH 保存形式
- 既存のルビ描画ルーチン本体
- フォルダ一括変換の基本処理
- プリセット保存の基本処理
- 既存の句読点・漢数字一・下鍵括弧補正の基本仕様

## 既知の注意

- PDF入力対応、PDF出力、CBZ出力はまだ含まれていません。
- 記号ペアの見え方はフォントによって異なるため、Noto 系フォントでは「縦中横記号」の下補正を必要に応じて使ってください。
